## ① 全体の動作確認（3分）

### 動作確認シナリオ

以下の流れを実際にやってみましょう：

#### シナリオ A：Q&A を登録してチャットで使われるか確認

1. **QA 管理タブ** を開く
2. 新しい Q&A を追加する：
   - カテゴリ：`programming`
   - 質問：「Next.js の App Router とは？」
   - 回答：「Next.js 13 以降で導入されたルーティングの仕組みです。ファイルベースで直感的にルートを定義できます。」
3. **チャットタブ** に切り替える
4. 「Next.js のルーティングについて教えて」と質問する
5. **確認ポイント：** 先ほど登録した Q&A の内容が AI の回答に反映されているか？

#### シナリオ B：関連のない質問をしてみる

1. チャットで「今日の天気は？」と質問する
2. **確認ポイント：** Q&A に関連データがないため、AI が一般的な回答をするか？

#### シナリオ C：会話の流れを確認

1. 「TypeScript とは何ですか？」と質問
2. 続けて「それを使うメリットは？」と質問
3. **確認ポイント：** 2つ目の質問で、前の会話（TypeScript について）を踏まえた回答になっているか？

### エラーが出たら

エラーメッセージをそのまま Claude Code に貼り付けて修正してもらいましょう：

> 「このエラーが出ました：（エラーメッセージを貼り付け）。直して」

### 研修全体のまとめ

#### 今日身につけたこと

| レベル   | スキル                                                     |
| -------- | ---------------------------------------------------------- |
| **基礎** | 自然言語でコードを生成する指示の出し方                       |
| **基礎** | ファイルの読み取り・編集・作成の委任                         |
| **中級** | エラーの解析・修正を Claude Code に依頼する方法              |
| **中級** | 既存コードベースの文脈を活用した指示                         |
| **実践** | DB 設計 → API → AI 統合 → UI の一連の開発フロー            |

#### Claude Code を使いこなすコツ

1. **指示は「友達に頼むように」自然な日本語で OK**
2. **範囲を絞って頼む** — 「全部作って」ではなく「まず○○だけ作って」
3. **エラーはそのまま貼り付ける** — 自分で読み解く必要はない
4. **わからないことはそのまま聞く** — 「これ何？」で教えてくれる
5. **やり直しを恐れない** — うまくいかなかったら「別のやり方で」と言えば OK

> **ゴール：** Step 1〜4 で学んだスキルを活かして、Claude Code に指示して **新しい AI 機能** を追加する

## やること

今のアプリではチャットで AI と会話できますが、**会話の内容を振り返る手段がありません**。

そこで、**「要約」ボタンを押すと AI が会話内容を要約してくれる機能** を追加します。

### 作るもの

1. **要約 API**（`POST /api/home/summary`）— 会話履歴を受け取り、Gemini で要約を生成して返す
2. **要約ボタン + 表示エリア** — チャット画面に追加

## ① 要約 API を作ろう（10分）

### 目標

会話履歴を受け取り、Gemini AI で要約して返す API を作ります。

### 期待されるリクエスト・レスポンス

**POST** `/api/home/summary`

リクエスト：

```json:features/home/routes/conversation.ts
{
  "messages": [
    { "role": "user", "content": "TypeScript って何ですか？" },
    { "role": "assistant", "content": "TypeScript は型安全な JavaScript です..." },
    { "role": "user", "content": "それを使うメリットは？" },
    { "role": "assistant", "content": "主なメリットは、バグの早期発見と..." }
  ]
}
```

レスポンス：

```json:features/home/routes/conversation.ts
{
  "summary": "TypeScript について質問があり、型安全な JavaScript であることと、そのメリット（バグの早期発見、IDE の補完機能など）について学びました。"
}
```

### 参考にできる既存コード

Step 3 で作った `features/home/routes/conversation.ts` が **そのままお手本** になります。この API も Gemini AI を呼び出してテキストを返す仕組みです。

Claude Code に「conversation.ts を参考にして」と伝えると、同じパターンで作ってくれます。

### まず自分で Claude Code に指示を出してみよう！

何を作りたいか、どこに作りたいかを考えて、**自分の言葉** で Claude Code に伝えてみましょう。

> 「チャットの会話履歴を要約する POST API を features/home/routes/summary.ts に作って。リクエストで messages（role と content の配列）を受け取って、Gemini に『以下の会話を日本語で3行以内で要約して』と送って、結果を返して。既存の features/home/routes/conversation.ts を参考にして。routes/index.ts にも登録して」

</details>

---

### 動作確認

API ができたら curl で確認しましょう：

```bash
curl -X POST http://localhost:3000/api/home/summary \
  -H "Content-Type: application/json" \
  -d '{"messages":[{"role":"user","content":"TypeScript って何？"},{"role":"assistant","content":"TypeScript は型安全な JavaScript です"}]}'
```

→ `{"summary": "..."}` のように要約テキストが返ってくれば API は完成！

> **うまくいかないとき：** エラーメッセージをそのまま Claude Code に貼り付けましょう。

## ② チャット画面に要約ボタンを追加しよう（12分）

### 目標

チャット画面に **「要約する」ボタン** を追加し、クリックすると会話の要約が表示されるようにします。

### やること

1. チャット画面に「要約する」ボタンを追加（会話が 2 件以上あるときだけ表示）
2. ボタンを押したら ① で作った API を呼び出す
3. 返ってきた要約をカード形式で画面に表示する

### まず自分で Claude Code に指示を出してみよう！

> **ポイント：** 一度に全部頼まず、**2 回に分けて** 指示するとうまくいきやすいです。
> まず「ボタンを追加して API を呼ぶ」、次に「見た目を整える」のように段階的に進めましょう。

### 講師の立ち回り

- **最初の 5 分** は全員に説明。その後は巡回サポート
- 自分で指示を考える時間を **最低 2 分** は与える。すぐにヒントを出さない
- 詰まっている人には「何を作りたいか日本語で説明してみて」と声をかける
- 進みが早い人には以下の追加チャレンジを提案：
  - 「要約を箇条書き形式にして」
  - 「要約の長さを選べるようにして（短い / 普通 / 詳しい）」
  - 「要約をクリップボードにコピーするボタンをつけて」

## トラブルシューティング

| 症状                          | 原因                          | 対処法                                                     |
| ----------------------------- | ----------------------------- | --------------------------------------------------------- |
| API が 404 になる              | ルートが登録されていない       | 「routes/index.ts に summary ルートが登録されているか確認して」 |
| 要約が英語で返ってくる         | プロンプトに日本語指定がない   | 「要約を日本語で返すようにプロンプトを修正して」               |
| ボタンを押しても反応しない     | API の URL が違う              | 「fetch の URL とルートのパスが一致しているか確認して」       |
| 要約が長すぎる                 | プロンプトの制約が弱い         | 「3行以内で要約するよう指示を強くして」と Claude Code に依頼  |
| 会話が少ないときにエラーが出る | メッセージ数のチェックがない   | 「メッセージが2件未満の場合はエラーを返すようにして」         |
